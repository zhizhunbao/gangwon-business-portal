# 1\. 1페이지 (메인 홈페이지) UI 수정 지시서

## 1.1 목적

Cursor 또는 Kiro가 빠르게 검색하고 정확히 수정할 수 있도록 변경 범위와 기준을 정리한다.

## 1.2 대상 화면

* 메인 홈페이지
* 상단 헤더 영역(로고, 메뉴바)
* 메인 콘텐츠 카드 섹션(섹션 제목)

## 1.3 변경 사항 요약

* 로고 교체
* 메뉴바 색상(로고 톤) 변경
* 전역 폰트(Pretendard) 변경
* 메뉴명 변경: 프로젝트 -> 사업공고
* 홈 섹션 제목 변경

  * 최신 공고 -> 사업공고
  * 뉴스 자료 -> 신규공고

## 1.4 로고 교체

### 1.4.1 요구사항

* 사이트 로고를 아래 이미지로 교체

  * https://k-talk.kr/static/images/user/logo.png

### 1.4.2 적용 위치

* 헤더 좌측 브랜드/로고 영역
* 로고가 여러 군데 쓰이면(로그인 상단, 사이드 헤더 등) 동일 이미지로 통일

  * 추후 별도 지시가 있으면 예외 처리

### 1.4.3 검색 힌트

* logo
* brand
* navbar-brand
* header-logo
* siteLogo
* assets/logo

## 1.5 메뉴바 색상 변경 (로고 톤과 동일 계열)

### 1.5.1 기준 색상

* 제공된 RGB: 0, 34, 68
* HEX 변환: #002244

### 1.5.2 요구사항

* 메뉴바 배경색을 #002244 로 변경
* 메뉴 글자/아이콘은 가독성 확보(흰색 계열 권장)
* hover/active는 같은 계열에서 살짝 밝은 색으로 적용
* 기존 스타일 시스템을 크게 바꾸지 말고, 가능하면 CSS 변수로 관리

### 1.5.3 참고

* \#10488c 는 RGB(16,72,140) 계열이라 RGB(0,34,68)과 톤 차이가 큼
* 로고 톤에 맞춘다는 목표면 #002244 를 우선 적용



## 1.6 폰트 변경 (Pretendard 계열)

### 1.6.1 요구사항

* 전역 폰트를 Pretendard 계열로 변경
* 적용 값

  * font-family: "pretendard", sans-serif !important;

### 1.6.2 적용 위치 권장

* html, body 전역
* UI 프레임워크/테마가 폰트를 덮어쓰는 경우 테마 설정도 함께 변경



## 1.7 메뉴명 변경

### 1.7.1 요구사항

* 메뉴명 변경

  * 프로젝트 -> 사업공고

## 1.8 홈 섹션 제목 변경 (카드 영역)

### 1.8.1 요구사항

* 최신 공고 섹션 제목

  * 최신 공고 -> 사업공고

* 뉴스 자료 섹션 제목

  * 뉴스 자료 -> 신규공고

### 1.8.2 검색 힌트

* 최신 공고
* 뉴스 자료
* section title
* widget title
* home cards

## 1.9 완료 기준(수용 조건)

* 헤더 로고가 지정 URL 이미지로 교체되어 표시됨
* 메뉴바 배경색이 #002244 로 적용됨
* 전역 폰트가 Pretendard, sans-serif 로 적용됨
* 메뉴 라벨이 사업공고로 표시됨(기존 프로젝트 대신)
* 홈 섹션 제목이 아래처럼 변경됨

  * 사업공고 (기존 최신 공고)
  * 신규공고 (기존 뉴스 자료)

* PC/모바일에서 레이아웃 깨짐 없음



# 2\. 2페이지 (기업정보) UI 수정 지시서

## 2.1 대상 화면

https://gangwon-portal-frontend.onrender.com/member/performance/company-info

* 기업정보 페이지
* 주소 정보 섹션
* 대표자 정보 입력 영역(대표자명 포함)

## 2.2 변경 사항 요약

* 대표자명 입력항목 아래에 출생년도(날짜) 추가
* 대표자명 입력항목 아래에 성별(남/여) 추가
* 소재지 항목을 화면 예시처럼 구성 및 추가

## 2.3 대표자 정보 입력 항목 추가 (출생년도, 성별)

### 2.3.1 추가 위치

* 대표자명 입력항목 바로 아래에 2개 필드 추가

  * 출생년도
  * 성별

### 2.3.2 성별 입력 방식

* 선택 방식: 남 / 여
* 권장 UI: 라디오 버튼 또는 셀렉트
* 값 저장 규칙

  * 화면 표시: 남, 여
  * 내부 값은 프로젝트 기존 규칙을 따른다

    * 예: M/F 또는 남/여 또는 0/1 등

  * 기존 API/DB 스키마가 있으면 그 규칙을 우선 적용

### 2.3.3 출생년도 입력 방식

* 달력(Date Picker)에서 날짜 선택
* 표시 및 저장 포맷

  * YYYY/MM/DD 형태로 표시
  * 저장 포맷은 프로젝트 기존 규칙을 따른다

    * 예: YYYY-MM-DD 또는 ISO string

* 필드명은 “출생년도”로 표기하되 실제 입력은 생년월일(YYYY/MM/DD) 단위



## 2.4 소재지 항목 UI 추가

### 2.4.1 요구사항

* 소재지 항목을 화면 예시처럼 드롭다운 선택 형태로 제공

### 2.4.2 드롭다운 옵션 목록

* 춘천시
* 원주시
* 강릉시
* 동해시
* 태백시
* 속초시
* 삼척시
* 홍천군
* 횡성군
* 영월군
* 평창군
* 정선군
* 철원군
* 화천군
* 양구군
* 인제군
* 양양군
* 기타 지역



### 2.4.3 동작 기준

* 현재 값이 1개만 선택되는 단일 셀렉트라면

  * 우선 단일 선택으로 구현
  * “소재지 추가”는 후속 기능 확장 포인트로 남겨도 됨

* 이미 여러 소재지를 추가하는 요구가 기존에 있다면

  * “소재지 추가” 영역에서 소재지를 추가하여 리스트로 누적되는 구조로 구현
  * 삭제/수정 UX는 기존 패턴을 따른다



## 2.5 완료 기준(수용 조건)

* 대표자명 바로 아래에 성별(남/여) 입력이 추가되어 저장 가능
* 대표자명 바로 아래에 출생년도 입력이 추가되어 달력으로 선택 가능
* 출생년도 표시 포맷이 YYYY/MM/DD 로 보임
* 소재지 항목이 화면 예시처럼 드롭다운 및 “기타 지역” 옵션을 포함
* “소재지 추가” 텍스트/영역이 화면에 표시됨
* PC/모바일에서 레이아웃 깨짐 없음

 

# 3. 3페이지 (성과입력) UI 수정 지시서
 
## 3.1 대상 화면
- 성과입력(성과 등록/수정) 페이지
- URL
  - https://gangwon-portal-frontend.onrender.com/member/performance/edit
- 탭 구조가 있는 입력 폼
  - 매출/고용 탭 내 수출액(원) 영역

## 3.2 변경 사항 요약
- 수출액(원) 입력 영역에 HSK 코드 입력 항목 추가
- 수출국가 입력 항목 추가(국가1, 국가2)
- HSK 코드 입력 규칙: 숫자 10자리
- HSK 코드 참고 링크 제공
  - https://cls.kipro.or.kr/hsk

## 3.3 추가 입력 항목 상세

### 3.3.1 추가 위치
- 매출/고용 탭
- 수출액(원) 섹션(전년도 / 해당연도 / 작성기준일 입력이 있는 줄 근처)
- 수출액(원) 섹션 바로 아래 또는 섹션 내부에 함께 배치
  - 화면 예시처럼 수출액(원) 라인과 자연스럽게 연결되도록 구성

### 3.3.2 HSK 코드 입력
- 라벨: HSK코드
- 입력값 규칙
  - 숫자만 허용
  - 정확히 10자리 입력
- 입력 UI
  - 기본 텍스트 입력(input)
  - 숫자 외 입력은 차단하거나 자동 제거
- 검증(Validation)
  - 10자리가 아니면 오류 표시 또는 저장 불가 처리(프로젝트 기존 검증 방식 준수)
- 도움말(Help)
  - [HSK코드] 참고 링크 노출
    - https://cls.kipro.or.kr/hsk

권장 UX
- placeholder 예시: 10자리 숫자 입력
- 입력 중에는 숫자만 유지(예: replace non-digit)
 

### 3.3.3 수출국가 입력 (국가1, 국가2)
- 라벨: 수출국가
- 입력 방식
  - 국가1, 국가2를 각각 입력할 수 있도록 2개 필드 제공
- 입력 UI 후보
  - 텍스트 입력 2개(가장 단순)
  - 또는 셀렉트/자동완성(프로젝트에 국가 목록 컴포넌트가 이미 있으면 재사용)
- 저장 규칙
  - 필드명/데이터 구조는 프로젝트 기존 성과 데이터 모델에 맞춘다
  - 아직 백엔드 필드가 없다면 프론트에서 임시로만 보관하지 말고, 연동 스펙 확인 필요
    - 단, 이번 문서에서는 UI 구현을 우선으로 한다

권장 UX
- placeholder 예시
  - 국가1 입력
  - 국가2 입력
 

## 3.4 텍스트 표기 요구사항
- 화면에 표시되는 문구(예시 기준)
  - HSK코드 : 숫자10자리 입력
  - 수출국가 : 국가1, 국가2 입력
  - [HSK코드] 참고 https://cls.kipro.or.kr/hsk
- 실제 화면 구성에서는 문구를 도움말/설명 텍스트로 적절히 배치

## 3.5 완료 기준(수용 조건)
- 성과입력 페이지의 수출액(원) 영역에 아래 항목이 추가됨
  - HSK코드 입력 필드
  - 수출국가 입력 필드 2개(국가1, 국가2)
- HSK코드는 숫자만 입력 가능하며 10자리 검증이 적용됨
- HSK코드 참고 링크가 화면에 노출됨
- 저장/임시저장 시 기존 기능이 깨지지 않음
- PC/모바일에서 레이아웃 깨짐 없음

- 회원 로그인으로 이동 클릭 시 404 페이지로 이동됨

### 10.4.2 요구사항
- 회원 로그인 페이지로 정상 이동하도록 링크 경로 수정
- 404가 발생하지 않도록 올바른 라우트로 연결

## 10.5 완료 기준
- 비밀번호를 잊으셨나요? 링크가 클릭되고 정상 이동함
- 회원 로그인으로 이동 링크 클릭 시 404가 발생하지 않고 회원 로그인 페이지로 이동함

# 11. 11페이지 (어드민: 내보내기 기능) 수정 지시서

## 11.1 대상 화면

- 관리자 > 실적 관리
- 관리자 > 프로젝트 관리(프로젝트 생성 메뉴 포함)

## 11.2 문제 요약

- 프로젝트 관리(프로젝트 생성 메뉴)에서 Excel 내보내기 / CSV 내보내기 실행 시
  - 파일이 아예 생성되지 않음

## 11.3 기업회원 관리: Excel, CSV 내보내기 파일 형식 오류
### 11.3.1 현상
- Excel 내보내기, CSV 내보내기 버튼 클릭 시 텍스트 파일로 다운로드됨

### 11.3.2 요구사항
- Excel 내보내기는 .xlsx 파일로 정상 다운로드
- CSV 내보내기는 .csv 파일로 정상 다운로드
- 파일명 규칙이 있다면 유지하고 확장자만 정상 적용
- 다운로드 시 Content-Type, 파일명, 인코딩이 올바르게 설정되어야 함

## 11.4 실적 관리: Excel, CSV 내보내기 파일 형식 오류
### 11.4.1 현상
- Excel 내보내기, CSV 내보내기 버튼 클릭 시 텍스트 파일로 다운로드됨

### 11.4.2 요구사항
- Excel 내보내기는 .xlsx 파일로 정상 다운로드
- CSV 내보내기는 .csv 파일로 정상 다운로드
- 기업회원 관리와 동일한 방식으로 수정하여 일관성 유지

## 11.5 프로젝트 관리(프로젝트 생성 메뉴): Excel, CSV 내보내기 파일 생성 안됨
### 11.5.1 현상
- Excel 내보내기, CSV 내보내기 버튼 클릭해도 파일이 다운로드되지 않음
- 다운로드 트리거가 동작하지 않거나, 응답이 없거나, 에러가 발생하는 상태로 추정

### 11.5.2 요구사항
- Excel 내보내기는 .xlsx 파일로 정상 다운로드
- CSV 내보내기는 .csv 파일로 정상 다운로드
- 버튼 클릭 시 반드시 다운로드가 시작되도록 수정

## 11.6 완료 기준
- 기업회원 관리에서 Excel, CSV 내보내기 결과가 각각 .xlsx, .csv로 정상 다운로드됨
- 실적 관리에서 Excel, CSV 내보내기 결과가 각각 .xlsx, .csv로 정상 다운로드됨
- 프로젝트 관리(프로젝트 생성 메뉴)에서 Excel, CSV 내보내기 버튼이 정상 동작하며 파일이 다운로드됨
- 동일한 내보내기 로직/유틸이 있다면 3개 화면이 동일 방식으로 동작하도록 통일됨


# 12. 12페이지 (어드민: 콘텐츠 관리 배너 이미지 규정 및 모바일 이미지 추가) 수정 지시서

## 12.1 대상 화면
- 관리자 > 콘텐츠 관리
- 배너 탭(메인 배너, 시스템 소개, 프로젝트, 성과, 지원 등 이미지 업로드 카드가 있는 화면)

## 12.2 변경 사항 요약
- 각 배너 항목에 모바일용 이미지 업로드 필드를 추가
- 웹에서 화면 폭이 태블릿 이하일 때 모바일 이미지를 우선 사용
- 모바일 이미지를 업로드하지 않은 경우에는 데스크탑 이미지를 그대로 사용
- 관리자 업로드를 위한 데스크탑/모바일 이미지 규격을 화면에 명시

## 12.3 업로드 필드 구조 변경
- 기존
  - 이미지 업로드 1개(데스크탑 이미지)
  - 링크 URL
- 변경
  - 데스크탑 이미지 업로드(기존 유지)
  - 모바일 이미지 업로드(신규 추가)
  - 링크 URL(기존 유지)

메모
- 모든 배너 카드(메인 배너/시스템 소개/프로젝트/성과/지원)에 동일하게 적용
- 저장 구조는 desktopImageUrl, mobileImageUrl 같은 형태로 분리 권장

## 12.4 데스크탑/모바일 이미지 규격(관리자 안내용)
공통 권장
- 파일 형식: JPG 또는 PNG
- 용량 권장: 1MB 이하(가능하면)
- 중요한 텍스트/로고는 중앙 영역에 배치(모바일에서 가장자리 잘림 방지)

### 12.4.1 데스크탑 이미지(권장)
- 권장 해상도: 1920 x 600
- 최소 해상도: 1440 x 450
- 권장 비율: 16:5 (가로형 배너)

### 12.4.2 모바일 이미지(권장)
- 권장 해상도: 1080 x 1350
- 최소 해상도: 750 x 938
- 권장 비율: 4:5 (세로형 배너)

메모
- 모바일은 세로형(4:5)로 두면 작은 화면에서 텍스트 가독성이 좋아짐
- 실제 배너 컴포넌트 높이가 고정이라면, 최종 배포 전 CSS의 배너 높이/비율과 맞춰서 수치 조정 가능
- “깨져 보임” 방지를 위해 업로드 시점에 리사이즈하지 말고, 표시 시 cover로 맞추는 방식 권장

## 12.5 웹에서 모바일 이미지 적용 규칙
### 12.5.1 적용 기준(브레이크포인트)
- 태블릿 이하 화면 폭에서 모바일 이미지 적용
- 권장 기준: 1024px 이하를 태블릿 이하로 간주

기본 규칙
- 화면 폭 <= 1024px 이고 모바일 이미지가 존재하면 모바일 이미지를 사용
- 모바일 이미지가 없으면 데스크탑 이미지를 사용

추가 메모
- 1024px 기준은 태블릿까지 포함하므로 “탭 이하 사이즈는 모바일 적용” 의도와 맞음
- 만약 태블릿에서는 데스크탑 이미지를 쓰고, 폰에서만 모바일 이미지를 쓰고 싶다면 768px 이하로 조정 가능
- 이번 요구사항은 “태블릿 사이즈 이하”에 모바일 이미지 적용이므로 1024px 이하 기준을 사용

### 12.5.2 표시 방식(CSS)
- 배너 이미지는 비율을 유지하고 cover로 채우기
- 찌그러짐 방지
  - object-fit: cover
  - object-position: center

## 12.6 어드민 UI 안내 문구(각 업로드 영역에 표시)
- 데스크탑 이미지
  - 권장: 1920 x 600 (16:5)
  - 최소: 1440 x 450
- 모바일 이미지(선택)
  - 권장: 1080 x 1350 (4:5)
  - 최소: 750 x 938
  - 모바일 이미지를 업로드하면 태블릿 이하(1024px 이하)에서 모바일 이미지가 표시됨
  - 모바일 이미지를 업로드하지 않으면 데스크탑 이미지가 그대로 표시됨

## 12.7 완료 기준(수용 조건)
- 콘텐츠 관리 화면에서 각 배너 항목마다 모바일 이미지 업로드가 가능함
- 모바일 이미지를 업로드하지 않아도 데스크탑 이미지가 모든 화면에서 정상 표시됨
- 모바일 이미지를 업로드한 경우, 화면 폭이 1024px 이하일 때 모바일 이미지가 표시됨
- PC/태블릿/모바일에서 이미지가 깨지거나 찌그러지지 않음
- 관리자 화면에 데스크탑/모바일 규격 안내가 명확히 표시됨
