# 강원창업포털 Frontend 수정 요청서

목적
- 아래 목록을 기준으로 Frontend UI/UX 및 기능 버그를 수정한다.
- 관리자용 페이지와 기업회원용 페이지를 구분하여 작업한다.
- 사이트 이름(브랜드명)을 강원창업포털 로 변경한다.

완료 기준 공통
- 저장이 안 되던 항목은 저장 후 새로고침, 재접속, 목록 재조회 시에도 값이 유지된다.
- 필수 항목은 비어있으면 저장 불가하며, 사용자에게 에러 메시지가 표시된다.
- 화면에 중국어가 노출되지 않는다.
- 날짜 포맷은 YYYY-MM-DD 로 통일한다.
- 배너 노출 조건이 요구사항과 일치한다.

1. 공통 레이아웃 전체 수정

요구사항
- 모든 페이지에서 센터 컨텐츠 영역 높이가 짧아 여백이 생기는 경우에도
  - 브라우저 전체 높이를 100%로 잡는다.
  - 푸터가 화면 하단에 붙어서 보이도록 한다.
- 스크롤이 필요한 페이지에서는 정상적으로 스크롤되며, 푸터는 컨텐츠 끝 다음에 위치한다.

체크 포인트
- 컨텐츠가 짧은 페이지: 푸터가 화면 하단에 붙어서 보임
- 컨텐츠가 긴 페이지: 푸터는 컨텐츠 끝에 위치, 스크롤로 접근

2. 사이트 이름 변경

요구사항
- 사이트 이름을 강원창업포털 로 변경
- 적용 위치 예시
  - 브라우저 탭 title
  - 헤더 로고 영역 텍스트
  - 로그인 화면 타이틀
  - 메타 정보(필요 시)
- 하드코딩된 기존 명칭이 있다면 전부 교체

3. 기업회원용 페이지

3.1 지원사업 페이지 기능 점검
작업 항목
- 페이지당 표시 기능 동작 체크
  - 페이지 사이즈 변경 시 목록 개수와 페이지네이션 정상 동작
- 상태 필터 동작 체크
  - 필터 변경 시 목록이 정상 필터링
- 검색 기능 동작 체크
  - 검색어 입력 후 조회 시 결과 반영
  - 검색 초기화 시 원복

3.2 실적관리 수정 페이지
URL
- https://gangwon-portal-frontend.onrender.com/member/performance/company-info

수정 요구사항
- 설립일 날짜 포맷 변경
  - 입력 및 표시 포맷: YYYY-MM-DD
  - 기존 데이터가 다른 포맷이면 화면에서 깨지지 않고 정상 표시되도록 처리
- 필수 입력 항목 검증 추가
  - 대표자명: 필수
  - 전화번호: 필수
  - 현재 비어있음 이슈 수정 필요
- 저장 불가 이슈 수정
  - 주요 사업 및 제품: 수정 후 저장 반영
  - 기업소개: 수정 후 저장 반영
  - 산업협력 희망 분야: 수정 후 저장 반영
- 조회 페이지에 항목 추가
  - 산업협력 희망 분야가 수정 페이지에는 있으나 조회 페이지에 없음
  - 조회 화면에도 동일 항목이 표시되어야 함
- 주소 정보 개선
  - 주소 검색 기능 추가
  - 현재 텍스트 입력만 가능
  - 요구 기능
    - 주소 검색 버튼 또는 입력 중 검색 트리거 제공
    - 검색 결과 선택 시 주소 필드 자동 채움
    - 상세주소는 별도 입력 가능 형태 권장

검증 시나리오
- 대표자명, 전화번호가 비어있을 때 저장 시도
  - 저장 불가
  - 필수 입력 안내 메시지 표시
- 주요 사업 및 제품, 기업소개, 산업협력 희망 분야 수정 후 저장
  - 저장 성공
  - 새로고침 후에도 값 유지
- 주소 검색으로 선택 후 저장
  - 조회 페이지에서 주소가 정상 표시

3.3 성과조회 페이지
URL
- https://gangwon-portal-frontend.onrender.com/member/performance/list

수정 요구사항
- 테이블 헤더 중국어를 한국어로 변경
  - 期间 -> 기간
  - 提交时间 -> 제출시간
  - 更新时间 -> 수정시간
- 성과조회에서 수정 시 특정 탭 데이터 저장 불가 문제 수정
  - 매출 고용 탭: 저장 반영
  - 정부지원 수혜 이력 탭: 저장 반영
  - 지식재산권 탭: 저장 반영

검증 시나리오
- 각 탭에서 값 변경 후 저장
- 목록으로 돌아가 재진입 또는 새로고침 후에도 데이터 유지 확인

3.4 최신공고 페이지 배너 누락
URL
- https://gangwon-portal-frontend.onrender.com/member/notices

수정 요구사항
- 최신공고 페이지에서 홈의 상단 배너가 표시되어야 한다.
- 현재 배너가 안 나오는 상태이므로 배너 노출 조건과 컴포넌트 연결을 수정

검증 시나리오
- 최신공고 진입 시 상단 배너 노출 확인
- 배너 스타일 및 위치 일관성 확인

3.5 프로젝트 상세보기 페이지 배너 및 담당자 표시
URL
- https://gangwon-portal-frontend.onrender.com/member/programs/b0fad89a-1374-4acf-bd23-2f259f8f7105

수정 요구사항
- 프로젝트 상세보기 페이지에서 프로젝트 메뉴의 배너가 보여야 한다.
- 프로젝트 신청 버튼 클릭 시 담당자 항목 데이터가 안 보이는 문제 수정
- 기업정보 페이지에 담당자 항목 추가
  - 대표자명 아래에 담당자 항목 추가
  - 입력, 저장, 조회 모두 가능해야 함
- 프로젝트 신청 화면에서 담당자 항목 표시
  - 기업정보에 저장된 담당자 데이터가 신청 화면의 담당자 영역에 표시되어야 함

검증 시나리오
- 기업정보에서 담당자 입력 및 저장
- 프로젝트 신청 화면에서 담당자 데이터 표시 확인

3.6 이용약관, 개인정보처리방침 노출
요구사항
- 기업회원 화면에서 이용약관, 개인정보처리방침은 관리자가 입력한 내용이 그대로 보여야 한다.
- 정적 텍스트가 아니라 콘텐츠 데이터 소스를 연동하여 표시

검증 시나리오
- 관리자에서 내용 수정 후
- 기업회원 화면에서 새로고침 후 반영 확인

4. 관리자용 페이지

4.1 콘텐츠 관리 버튼명 한글화
URL
- https://gangwon-portal-frontend.onrender.com/admin/content

수정 요구사항
- 콘텐츠 관리에서 预览 버튼명을 한국어로 변경
  - 预览 -> 미리보기

검증 시나리오
- 해당 화면에서 버튼명이 한국어로 노출되는지 확인

4.2 이용약관, 개인정보처리방침 관리자 입력 기능 추가
요구사항
- 관리자에서 이용약관, 개인정보처리방침을 클릭하면 에디터 창으로 내용 입력 가능해야 함
- 최소 기능
  - 에디터 열기
  - 내용 입력
  - 저장
  - 저장 후 다시 열었을 때 저장된 내용 로드
- 기업회원 화면 노출과 동일 데이터 소스를 사용해야 함

검증 시나리오
- 이용약관 내용 입력 후 저장
- 개인정보처리방침 내용 입력 후 저장
- 각각 재접속 후에도 내용 유지 확인
- 기업회원 화면에서 동일 내용 노출 확인

4.3 기업회원관리에서 담당자 항목 조회 추가
요구사항
- 관리자 기업회원관리 페이지에서도 담당자 항목이 조회되도록 추가
- 기업회원이 입력한 담당자 데이터가 관리자 조회 화면에 표시되어야 함

검증 시나리오
- 기업회원에서 담당자 입력 후 저장
- 관리자 기업회원관리에서 해당 기업 조회 시 담당자 값 표시 확인

작업 체크리스트
