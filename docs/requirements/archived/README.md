# 客户输入文档

## 📋 目录说明

本目录用于存放客户提供的原始需求文档，包括PPT、设计稿、需求文档等。

## 📁 目录结构

```
customer-inputs/
├── README.md                    # 本说明文件
├── active/                      # 当前处理中的需求
│   ├── 2025-01-25-customer-a/   # 按日期和客户分类
│   │   ├── requirements.pptx    # 客户PPT文档
│   │   ├── analysis.md          # 需求分析报告
│   │   └── prototype/           # 原型文件
│   └── 2025-01-26-customer-b/
├── completed/                   # 已完成的需求
│   ├── 2024-12-15-project-x/
│   └── 2024-12-20-project-y/
└── archived/                    # 已归档的需求
    └── 2024-q1/
```

## 📂 文件命名规范

### 目录命名
```
YYYY-MM-DD-customer-name/
```

**示例**:
- `2025-01-25-gangwon-enterprise/`
- `2025-01-26-startup-inc/`
- `2025-01-27-government-dept/`

### 文件命名
```
customer-name_document-type_version.extension
```

**示例**:
- `gangwon-enterprise_requirements_v1.pptx`
- `startup-inc_ui-design_v2.fig`
- `gov-dept-functional-spec_v1.docx`

## 🔄 处理流程

### 1. 接收文档
- 将客户文档放入 `active/` 目录
- 按日期和客户创建子目录
- 重命名文件符合命名规范

### 2. 需求分析
- 使用产品经理技能分析PPT
- 生成需求分析报告
- 创建原型设计

### 3. 客户确认
- 与客户确认需求
- 记录客户反馈
- 更新分析报告

### 4. 技术转化
- 转化为技术需求文档
- 移动到 `completed/` 目录
- 开始开发工作

### 5. 项目归档
- 项目完成后移动到 `archived/`
- 按季度或年份归档

## 📋 文档管理

### 版本控制
- 每次更新文档时更新版本号
- 保留历史版本
- 记录变更内容

### 备份策略
- 定期备份客户文档
- 使用云存储备份
- 确保文档安全

### 访问权限
- 客户文档仅限项目团队访问
- 敏感信息加密存储
- 定期审查访问权限
